# Implementation Plan: Sprint S0 — Scaffold

## Goal

Bootstrap the SNY Planner project from an empty directory into a running Next.js 14
application with: TypeScript, Tailwind CSS, App Router, a Prisma schema connected to
Neon PostgreSQL, an empty-but-structured page shell (Orders / Schedule / Materials),
a seed file with 5 sample production orders, and a correct `.gitignore`. No business
logic. No authentication. No real data calculations. Just the skeleton Tung can push
to GitHub and hand off to the next sprint.

> **⚠️ Note:** `CLAUDE.md` was NOT found in the project root at plan time. This plan
> is based entirely on the prompt. If Tung has a `CLAUDE.md` with a different DB
> schema or folder structure, **share it before approving this plan** so it can be
> incorporated. CLAUDE.md always wins over this prompt.

## Steps

1. **Init Next.js project** — Run `npx create-next-app@14 . --typescript --tailwind --app --src-dir --import-alias "@/*" --use-npm --no-eslint` in the project root. Verify `package.json`, `tsconfig.json`, `tailwind.config.ts`, and `src/app/` are created.

2. **Install Prisma** — Run `npm install prisma tsx --save-dev` and `npm install @prisma/client`. Then add `"prisma": { "seed": "tsx prisma/seed.ts" }` to `package.json`.

3. **Write `prisma/schema.prisma`** — Define the `ProductionOrder` model with fields: `id`, `piNumber`, `subLineIndex`, `customer`, `orderDate`, `widthM`, `lengthM`, `gsm`, `color`, `status`, `createdAt`, `updatedAt`. Add Neon-compatible datasource block and `generator client`.

4. **Write `src/lib/db.ts`** — Prisma singleton pattern using `globalThis` to prevent hot-reload duplication in Next.js dev mode.

5. **Write `.env.example` and `.env.local`** — `.env.example` contains placeholder `DATABASE_URL`. `.env.local` is created as an empty file for Tung to fill with the Neon connection string.

6. **Write `prisma/seed.ts`** — 5 sample orders using `upsert` keyed on `piNumber+subLineIndex` (idempotent). Includes 1 piNumber with 2 sub-lines (subLineIndex 0 and 1). Realistic SNY data: widths 3.2–4.5m, gsm 140–200, colors BLACK/WHITE/BLUE etc.

7. **Write root layout (`src/app/layout.tsx`)** — HTML skeleton with `<head>` meta tags, top navigation bar with links: Orders | Schedule (MOCK) | Materials (MOCK). Tailwind-styled. Imports `globals.css`.

8. **Write page shells** — Four files:
   - `src/app/page.tsx` — server-side `redirect('/orders')` from `next/navigation`
   - `src/app/orders/page.tsx` — "Production Orders" heading + "New Order" button (non-functional)
   - `src/app/schedule/page.tsx` — heading + MOCK banner
   - `src/app/materials/page.tsx` — heading + MOCK banner

9. **Write `.gitignore`** — Ensure it includes: `.env.local`, `.env*.local`, `node_modules/`, `.next/`, `*.log`.

10. **Run `npx prisma generate`** — Generates Prisma client types from schema. Does NOT connect to DB (safe without `DATABASE_URL`). Verify `node_modules/@prisma/client` exists.

## Risks

1. **`create-next-app@14` may scaffold Next.js 15** — Latest create-next-app may default to Next.js 15.
   - Mitigation: Explicitly pin `create-next-app@14`. Verify `"next": "14.x.x"` in generated `package.json`. If it installs 15, stop and notify Tung.

2. **Prisma 7.x API changes** — Current stable is 7.8.0; some online references show 5.x patterns. Core APIs (`PrismaClient`, `upsert`, `$disconnect`) are stable across versions.
   - Mitigation: Use only documented stable APIs. Flag any uncertainty.

3. **`prisma generate` may warn about missing `DATABASE_URL`** — `generate` reads schema but does NOT connect to DB. Should succeed without `.env.local`. Some versions print a warning.
   - Mitigation: Document that the warning is safe to ignore. If the command exits with an error (not warning), skip it and note in the build summary.

4. **Windows + OneDrive non-ASCII path** — The project path contains Vietnamese characters (`Máy tính`). `npx` and `create-next-app` may fail with this path.
   - Mitigation: Test the scaffold command first. If it fails with a path error, stop immediately and ask: "Please move the project to `C:\dev\sny-planner` to avoid Unicode path issues."

5. **`tsx` seed runner requires devDependency installation** — `npx prisma db seed` calls `tsx` via `node_modules/.bin/tsx`. If `tsx` is not installed, seed will fail.
   - Mitigation: Install `tsx` as devDependency explicitly. Verify it appears in `devDependencies` in `package.json`.
